Module 4
Application Layer
Apply the what you learned about ports towards a real-world example.

In this module, we’ll cover the HTTP application layer. This will give you a brief example of its purpose and how it plays into the innerworkings of the internet. These concepts are similar to HTTPS, SMTP and FTP. The way messages start off and its tone of voice with each application layer is different and lives at different house numbers.

HTTP

HTTP (Hypertext Transfer Protocol) is a way messages are structured on the internet. For example, you type in “facebook.com” in Google Chrome and press “enter”. Your browser writes a letter to Facebook Street. As the mailman gets to Facebook Street, he needs to deliver the letter to a house. The house we deliver it to is #80 because on the letter, the address is written as “HTTP” and the mailman already knows that HTTP occupies house #80. What you need to understand is that house #80 is actually what we call port 80.

Since Facebook’s HTTP lives at port 80, it just waits until something is sent to Facebook Street at port 80. An HTTP web server will patiently wait for its requests (letters) at its home on port 80. If you write a request (letter) to Facebook Street on port 80, you will always get something back.

Once this request (letter) is received, the web server sends back a message like “HTTP/1.1 200 OK” and writes the file you requested in the body of the letter, which is commonly a webpage in HTML format.
